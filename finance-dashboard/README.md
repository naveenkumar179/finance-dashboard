# Finance Dashboard API

A robust backend API for managing financial records and generating dashboard analytics with strict Role-Based Access Control (RBAC). Built with Spring Boot, this application provides secure endpoints for financial data management and real-time insights.

## Features

### Security
- Role-Based Access Control (RBAC) with JWT authentication
- Permissions for Admin, Analyst, and Viewer roles

### Data Management
- CRUD operations for financial records (income/expense)
- Soft deletion for records
- Filtering by category, date, and type

### Analytics
- Dashboard summaries with totals, balances, and trends
- Category-wise and monthly aggregations

### Additional Features
- Rate limiting with Bucket4j
- Swagger API documentation
- Automatic database schema generation with Hibernate

## Technologies Used

- **Java 17**
- **Spring Boot 3.x**
- **Spring Security** (JWT Authentication)
- **Spring Data JPA** (Hibernate)
- **MySQL Database**
- **Bucket4j** (Rate Limiting)
- **Swagger/OpenAPI** (API Documentation)
- **Lombok** (for generating getters, setters, constructors, etc.)
- **Maven** (Build Tool)

## Prerequisites

- Java 17 or higher
- MySQL Server
- Maven 3.6+

## Installation

1. **Database Setup**
   - Configure MySQL database connection in `src/main/resources/application.properties` with the credentials. 
     ```properties
     spring.datasource.url=jdbc:mysql://localhost:3306/finance_db
     spring.datasource.username=finance_user
     spring.datasource.password=finance123
     ```
   - **Note**: The hostname is typically `127.0.0.1` (localhost) and the default port is `3306` for MySQL.

2. **Build the Application**
   ```bash
   mvn clean install
   ```

3. **Run the Application**
   ```bash
   mvn spring-boot:run
   ```

   The application will start on `http://localhost:1002/swagger-ui/index.html`. Hibernate will automatically create the required database tables.

**Steps to Authenticate via Swagger:**
1. Access Swagger UI at: `http://localhost:1002/swagger-ui/index.html`
2. Navigate to `POST /auth/login` and execute with email/password.
3. Copy the `token` from the response.
4. Click the **Authorize** button and paste the token (do not type "Bearer " before it).

**Note**: JWT tokens expire after 1 hour. You will need to re-authenticate after expiration.

## API Endpoints

### Authentication
- `POST /auth/login`: Authenticate and receive JWT token (Open to all).

### Users
- `POST /users`: Create new user (Admin only).
- `GET /users`: Get all users (Admin only).
- `PUT /users/{id}`: Update user (Admin only).
- `PUT /users/{id}/status`: Update user status (Admin only).

### Dashboard
- `GET /dashboard/summary`: Get aggregated financial summaries (Admin, Analyst, Viewer).

### Financial Records
- `GET /records`: Paginated list of active records (Admin, Analyst).
- `GET /records/category/{category}`: Records by category (Admin, Analyst).
- `GET /records/date/{date}`: Records by date (Admin, Analyst).
- `GET /records/type/{type}`: Records by type (Admin, Analyst).
- `POST /records`: Create new record (Admin only).
- `PUT /records/{id}`: Update record (Admin only).
- `DELETE /records/{id}`: Soft delete record (Admin only).

## Architecture

The application follows a layered architecture:

```
src/main/java/com/finance/dashboard/
├── config/          # Configuration classes (Security, Swagger, Rate Limiting)
├── controller/      # REST controllers and request handling
├── model/           # JPA entities and enums
├── repository/      # Data access layer
├── security/        # JWT utilities and filters
├── service/         # Business logic layer
├── exception/       # Global exception handling
└── view/            # JSON view projections
```
